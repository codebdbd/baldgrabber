# Правила сборки релиза BaldGrabber

Этот файл обязателен к прочтению перед любой локальной сборкой, упаковкой или публикацией релиза.

## Постоянная структура

- `BaldGrabber_Release/` — единственная актуальная self-contained publish-сборка приложения. Установщик собирается только из неё.
- `BaldGrabber_Portable/` — единственная актуальная распакованная portable-версия.
- `BaldGrabber_Portable.zip` — единственный актуальный архив portable-версии.
- `Output/BaldGrabber_Installer.exe` — единственный актуальный установщик.

Нельзя создавать в корне или внутри проекта дополнительные каталоги `artifacts`, `verification`, `publish`, `release-stage`, `local-publish`, `tool-temp` и другие варианты релизных или проверочных папок.

## Временные файлы

- Для промежуточных проверок использовать только `%TEMP%\BaldGrabber\...`.
- После проверки удалять созданные временные файлы.
- Никогда не называть временную сборку актуальным релизом.

## Обязательная последовательность локального релиза

1. Проверить, что версия в `BaldGrabber/BaldGrabber.csproj` совпадает с `AppVersion` в `BaldGrabber_Installer.iss`.
2. Выполнить Release-сборку.
3. Выполнить self-contained publish непосредственно в `BaldGrabber_Release/`, обновив её содержимое.
4. Проверить версию и запуск `BaldGrabber_Release/BaldGrabber.exe`.
5. Обновить приложение внутри существующей `BaldGrabber_Portable/` и пересобрать существующий `BaldGrabber_Portable.zip`.
6. Собрать установщик из `BaldGrabber_Release/` в `Output/BaldGrabber_Installer.exe`.
7. Проверить версии, время изменения и наличие обязательных файлов во всех трёх вариантах релиза.

## Запреты

- Не менять структуру каталогов без прямого запроса пользователя.
- Не создавать альтернативное место для «актуального релиза».
- Не оставлять старую версию в `BaldGrabber_Release/` после заявления об успешной сборке.
- Не удалять и не переименовывать релизные каталоги без прямого запроса пользователя.
- Не менять номер версии без явного согласования.
- Не коммитить, не создавать тег и не публиковать GitHub Release без прямого запроса пользователя.

## Критерий готовности

Релиз считается собранным только тогда, когда `BaldGrabber_Release`, `BaldGrabber_Portable`, `BaldGrabber_Portable.zip` и `Output/BaldGrabber_Installer.exe` обновлены одной версией и проверены. Обычная компиляция в `bin/` релизом не считается.
