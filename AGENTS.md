# AGENTS.md

## Правила работы с проектом BaldGrabber

Перед любой сборкой, упаковкой или публикацией релиза обязательно полностью прочитать и выполнить `RELEASE_RULES.md` из корня проекта.

### Обязательная сборка после изменений
- После любых изменений в проекте всегда выполнять полный релизный цикл по `RELEASE_RULES.md`.
- Актуальную self-contained сборку всегда публиковать непосредственно в `BaldGrabber_Release/`.
- Одновременно обновлять `BaldGrabber_Portable/`, `BaldGrabber_Portable.zip` и `Output/BaldGrabber_Installer.exe`.
- Сборки из `bin/`, `obj/` и временных каталогов никогда не считать релизом и никогда не предлагать пользователю для проверки.
- Для ручной проверки всегда указывать только `BaldGrabber_Release/BaldGrabber.exe`.

### Обязательное обновление списка возможностей
- При добавлении, удалении или изменении поддерживаемых источников, типов ссылок, форматов, качеств или вариантов загрузки всегда обновлять корневой файл `DOWNLOAD_CAPABILITIES.md`.
- В `DOWNLOAD_CAPABILITIES.md` перечислять только фактически реализованные и проверенные возможности программы.

### Запрещено
- Не ломать то, что работает. Если тебя попросили изменить одну строку — меняй одну строку, а не переписывай половину файла
- Не добавлять fallback-и которые молча заменяют выбор пользователя. Если формата нет — скажи, не подменяй
- Не менять логику скачивания без явной просьбы
- Не удалять код который работает, просто чтобы "упростить"
- Не трогать конвертеры, сервисы настроек, модели данных если в этом нет необходимости
- Не спрашивать "а что если сделать лучше?" — делай то что просят

- Исправлять конкретные баги если указывают
- Менять текст/лейблы/локализацию
- Менять стили/дизайн если просят

### Технологии
- .NET 9, WinUI 3 (WinAppSDK)
- CommunityToolkit.Mvvm
- yt-dlp + ffmpeg для скачивания
- Серийная логика: URL → yt-dlp → файл
- Локализация: RU, EN, UK

### Структура проекта
- `ViewModels/MainViewModel.cs` — основная логика
- `Services/DownloadService.cs` — скачивание через yt-dlp
- `MainPage.xaml` — UI
- `MainPage.xaml.cs` — code-behind
- `Models/` — модели данных
- `Converters/` — конвертеры XAML
